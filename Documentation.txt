ABSTRACT
AgriNova is a mobile-centric digital advisory system developed to support practical
agricultural decision-making in Nepal. The system addresses frequent challenges in
traditional farming such as uncertain crop selection, inconsistent use of soil data,
and difficulty finding accessible agricultural markets. The platform integrates two
core intelligent modules: a crop recommendation engine and a nearest market finder.

The backend is implemented in Django REST Framework with token-based
authentication and modular services for user management, crop recommendation,
and market search. The frontend is implemented in Flutter and connects to REST APIs
for registration, profile handling, recommendation requests, recommendation history,
market search, and admin operations. Crop recommendation is generated using a
Decision Tree based approach over soil parameters including pH, nitrogen,
phosphorus, potassium, rainfall, and season. Nearest market discovery is performed
using the Haversine formula over latitude/longitude coordinates.

The system currently supports role-based workflows for both users and administrators.
Users can submit soil data, receive ranked recommendations, track recommendation
history, and find nearby markets. Administrators can manage users, crop records,
markets, and market prices through protected endpoints. The solution is designed in
an iterative Agile workflow, allowing incremental delivery and improvement.

By combining explainable recommendation logic and location-aware market guidance
in one platform, AgriNova contributes to better farm planning, improved accessibility
to market channels, and more reliable data-driven decisions.

Keywords: AgriNova, Agriculture Advisory, Decision Tree, Haversine Formula,
Django REST Framework, Flutter, Crop Recommendation, Market Finder,
Token Authentication, Agile Methodology.

CHAPTER 1: INTRODUCTION

1.1 Introduction
AgriNova is an advanced digital agriculture advisory platform designed to improve farming
decision-making for both individual farmers and agricultural administrators. The system is
developed to provide a practical and easy-to-use environment where users can make
evidence-based choices about crop planning and market access instead of depending only
on traditional experience or guesswork.

For users, the platform provides a clear workflow that starts from account access and data
entry. Farmers can register, log in, manage their profile, and submit soil-related parameters
such as pH, nitrogen, phosphorus, potassium, rainfall, district, and season. Based on these
inputs, the system generates ranked crop recommendations using the Decision Tree based
recommendation engine. Users can also review previous recommendation records, which
supports continuity in planning across multiple farming cycles.

In addition to crop recommendation, AgriNova includes a nearest market finder module.
This module allows users to provide location coordinates and obtain nearby agricultural
markets ordered by distance. By combining crop advisory and market discovery in one
system, AgriNova helps reduce the gap between production planning and produce selling,
which is a common challenge in practical farming workflows.

Once authenticated, users can interact with a complete set of advisory services from a
single mobile application interface. The system stores recommendation history and market
search history for later reference, helping users compare outcomes and make more informed
future decisions. For new users and existing users alike, this creates a structured advisory
experience rather than isolated one-time calculations.

The system is organized into two major operational roles: User and Admin. In the User
role, farmers can perform profile management, soil-data submission, recommendation
retrieval, and nearest market search. In the Admin role, authorized administrators can
manage core datasets through CRUD operations, including users, crop records, market
records, and market prices. The platform follows a client-server architecture with Flutter on
the client side and Django REST Framework on the server side, enabling secure token-based
authentication, role-based authorization, and reliable API-driven communication.

1.2 Problem Statement
In many farming communities, crop decisions are still made mainly from past experience,
local assumptions, or informal advice. While traditional knowledge remains valuable, it is
often insufficient when soil quality differs from one location to another or when weather and
market conditions change quickly. As a result, farmers may cultivate crops that are not well
matched to their soil profile, which can reduce yield and increase production risk.

Resource planning challenges further increase this problem. Without a structured
data-supported approach, fertilizer and input use can become imbalanced, either excessive
or insufficient, affecting both cost and soil health. In addition, many farmers face difficulty
in identifying suitable nearby markets at the right time, which can lead to delayed sales,
higher transport burden, and reduced profitability.

Another major issue is the lack of a unified digital workflow that connects crop advisory
with market accessibility and historical record-keeping. When recommendation records and
market search history are not preserved in one system, users cannot easily compare past
decisions or improve future planning. This creates repeated uncertainty in each farming
cycle.

Therefore, a practical and mobile-friendly advisory platform is required to automate
soil-based crop recommendation, provide location-aware nearest market guidance, and
maintain decision history in a secure and structured way. AgriNova is developed to address
these exact needs and to support more reliable, data-driven agricultural planning.

1.3 Objectives
The objectives of this project are:
To provide personalized crop recommendations based on soil and seasonal data.
To help farmers find the nearest agricultural markets quickly.
To maintain recommendation and market search history for better future decisions.

1.4 Scope and Limitation

1.4.1 Scope
This project involves developing a mobile-based agriculture advisory system using modern,
open-source technologies, including Flutter for the frontend, Django REST Framework for
the backend, and SQLite in the current development setup for data storage. The system
enables users to register, log in, manage profiles, submit soil and seasonal parameters,
receive personalized crop recommendations, and find nearby agricultural markets using
location-based search. The platform also maintains recommendation and market-search
history so users can review previous records for future planning. Additionally, the system
includes an admin management module that allows authorized administrators to manage
users, crop records, market records, and market price information through role-protected
CRUD operations.

1.4.2 Limitations
The system's performance may be impacted under high traffic conditions because the
current implementation is primarily focused on core functionality and does not yet include
advanced production-level optimization and scaling strategies.
The project currently uses a development-oriented setup, and production hardening features
such as strict CORS policies, dedicated deployment architecture, and advanced forecasting
modules are not included in this version.

CHAPTER 2: BACKGROUND STUDY AND LITERATURE REVIEW

2.1 Background Study
Digital agriculture advisory systems are technology platforms that help farmers improve
planning by using data rather than only traditional assumptions. In Nepal, adoption of
mobile-based agriculture support has gradually increased with wider smartphone access,
better internet connectivity, and growing interest in digital services in rural and semi-urban
areas. Several platforms and services now provide guidance related to farming practices,
weather updates, and agricultural information, helping users access support from their
devices instead of relying only on manual channels.

In practical use, however, many available solutions focus on only one part of the farming
workflow. Some tools provide general advisory content but do not offer personalized crop
recommendation based on soil parameters. Others provide market-related information but
do not connect it directly with crop planning decisions. Because of this separation, farmers
often need multiple tools to complete one decision cycle, from selecting a suitable crop to
finding a nearby market for selling produce.

AgriNova is developed to address this gap through an integrated advisory approach. The
system combines a Decision Tree based crop recommendation module and a location-based
nearest market finder in a single account-based platform. Users can submit soil and seasonal
data, receive ranked crop recommendations, and then identify nearby markets through the
same application. By unifying recommendation, market discovery, and history tracking,
AgriNova provides a more complete and user-friendly digital agriculture workflow for the
current context of Nepal.

2.2 Literature Review
The development of digital agriculture advisory systems has significantly influenced modern
farming practices in many countries, including Nepal. These platforms generally provide
users with access to agronomic tips, crop information, weather updates, and market-related
content through mobile and web channels. However, many currently available systems focus
mainly on general information delivery and do not provide a unified, personalized workflow
that connects soil-based crop recommendation with nearest market guidance in one place.

AgriNova addresses this gap by combining two practical decision-support components: a
Decision Tree based crop recommendation module and a location-aware nearest market
finder. The recommendation module provides ranked crop suggestions for authenticated
users based on soil and seasonal input values, while the market module helps users locate
nearby agricultural markets through coordinate-based distance calculation. In addition,
AgriNova stores recommendation and market-search history so users can compare previous
results and make better decisions over time.

Plantix
Plantix is a widely known agriculture application that supports farmers through image-based
pest and disease detection. It provides useful advisory content for crop health management
and has contributed to digital awareness in agriculture. However, its primary focus is on
disease diagnosis and guidance, and it does not specifically provide integrated
soil-parameter crop recommendation with nearest market discovery in a unified workflow.

Krishi Guru
Krishi Guru is a Nepal-focused agriculture information platform that offers farmers practical
knowledge on crop cultivation and related activities. It is useful for awareness and
educational support, especially for users seeking general farming guidance. Still, the
platform does not strongly emphasize personalized recommendation using soil chemistry and
seasonal parameter inputs with trackable recommendation history.

AgriApp and Similar Advisory Platforms
AgriApp and related tools provide weather information, crop guidance, and agri-input
support, which are valuable for routine farming decisions. Despite these strengths, many
such systems are feature-specific and do not fully integrate account-based recommendation,
history management, and nearest-market retrieval under one coherent advisory flow.

Therefore, the reviewed systems demonstrate the usefulness of digital agricultural services,
but also highlight a practical need for integrated, data-driven, and user-specific support.
AgriNova is designed to meet this need by combining recommendation, market access, and
historical decision tracking in a single mobile-friendly platform.

3.1 Analysis System
System analysis is the process of examining an existing domain and proposed workflow to
understand system components, identify practical issues, and define requirements for a
reliable software solution. It includes requirement gathering, process understanding, data
identification, and objective definition to guide architecture and implementation. In this
project, system analysis was essential to understand how AgriNova should support farmers
and administrators through data-driven crop recommendation, nearest market search,
history tracking, and controlled data management.

Types of System Analysis
Structured Approach
Breaks down the system into process-oriented components and emphasizes data flow.
Uses process-focused tools like DFDs, ER models, and flowcharts.
Works well for strictly procedural implementations.

Object-Oriented Approach (OOP)
Represents the system as interacting objects that combine attributes and behavior.
Uses UML models such as use case, class, sequence, activity, component, and deployment
diagrams.
Supports modularity, reusability, extensibility, and clean maintenance.

Reason for Choosing Object-Oriented Approach
The Object-Oriented Approach was selected for AgriNova because it aligns with the current
technology stack and architecture. Flutter uses component/widget-based structure and
Django follows object-oriented models, serializers, and views. Core entities such as User,
Profile, Crop, SoilData, Recommendation, Market, and MarketPrice can be naturally modeled
as software objects. This improves readability, maintainability, and future scalability while
allowing clear separation of concerns between modules.

3.3 Algorithm Details

Decision Tree Algorithm (Crop Recommendation)
The Decision Tree based approach is used to classify suitability of crops from soil and
seasonal parameters. The engine evaluates pH, nitrogen, phosphorus, potassium, rainfall,
and season to generate ranked recommendations.

Mathematical formulation used conceptually:
Entropy:
H(S) = -sum(pi * log2(pi))

Information Gain:
G(S,A) = H(S) - sum((|Sv| / |S|) * H(Sv))

Working Steps:
Input collection from user soil and seasonal fields.
Validation and normalization of input payload.
Engine evaluation and score generation.
Ranking of suitable crops.
Persistence of recommendation and ranked scores.
Return of ranked results with confidence score.

Haversine Formula (Nearest Market Finder)
The nearest market module computes great-circle distance from user coordinates to market
coordinates and returns markets ordered by proximity.

Formula:
d = 2r * arcsin(sqrt(sin^2(delta_lat/2) + cos(lat1)*cos(lat2)*sin^2(delta_lon/2)))

where:
d = distance between two points
r = Earth's radius (about 6371 km)
lat1, lat2 = latitude values
lon1, lon2 = longitude values

Working Steps:
Receive latitude, longitude, and optional district filter.
Retrieve active markets from database.
Compute distance for each market.
Sort by shortest distance and limit by requested max results.
Store search history and return ranked market list.


CHAPTER 4: IMPLEMENTATION AND TESTING

4.1 Implementation
AgriNova works as a mobile-based advisory application with a Flutter frontend and a
Django REST backend. The frontend is designed to provide a simple and accessible user
experience for farmers and administrators, while the backend handles authentication,
business logic, algorithm execution, and persistent data management. API communication is
performed through token-based authorization, and endpoints are grouped into dedicated
modules for authentication, crop recommendation, and market finder operations.

For current development and testing, the backend uses SQLite as the default database.
This setup supports rapid development and verification of features such as profile updates,
soil-data submission, recommendation generation, market search, and history tracking.
Swagger and ReDoc documentation routes are also integrated to test and verify API behavior
during implementation.

As planned in earlier design phases, the project follows Agile methodology. Core modules
were implemented incrementally, tested in short cycles, and improved iteratively. This
approach allowed continuous validation of both user-facing and admin-facing workflows,
ensuring that the system remained consistent, modular, and extensible.

The implemented system allows users to register, log in, submit soil and seasonal input,
receive ranked crop recommendations, find nearest markets by coordinates, and view
historical records. On the administrative side, protected management features are available
to maintain crop, market, market-price, and user records. The final implementation provides
an end-to-end advisory flow from data entry to actionable recommendation and market
discovery.

4.1.2 Implementation details of modules
In AgriNova, implementation is organized around two main operational roles: Admin and
User. Each role has specific features aligned with system requirements.

Admin Module
The Admin Module is responsible for maintaining core datasets and controlled system
operations. Administrators ensure that the platform has valid crop and market data and can
perform management actions through protected endpoints.

The admin can perform the following actions:
Login
Admin users authenticate using valid credentials to access secured management functions.

Manage Users
Admins can view users and delete non-admin user accounts through role-restricted APIs.

Manage Crops
Admins can add, update, and remove crop records, including agronomic attributes used by
the recommendation engine.

Manage Markets
Admins can add, update, and remove market records with district and coordinate data.

Manage Market Prices
Admins can create and maintain market-specific crop price entries.

Logout
Admins can terminate sessions securely after completing management tasks.

User Module
The User Module provides advisory functionality for regular users. It supports account
access, input submission, recommendation retrieval, market search, and historical review.

Users can perform the following actions:
Register / Login
Users create accounts and authenticate to access personalized advisory features.

Profile Management
Users can view and update profile and extended profile details.

Submit Soil Data and Get Recommendation
Users submit pH, NPK, rainfall, district, and season values to receive ranked crop
recommendations generated by the Decision Tree based engine.

View Recommendation History
Users can access previous recommendation records and details for decision continuity.

Find Nearest Markets
Users submit location coordinates to receive nearby markets ordered by distance, with
district-based filtering when required.

View Market Search History
Users can review previously executed market searches.

Logout
Users can log out after using the advisory services to maintain account security.

CHAPTER 5: CONCLUSION AND FUTURE RECOMMENDATIONS

5.1 Conclusion
The AgriNova system developed in this project successfully fulfills its primary objectives.
It enables users to submit soil and seasonal information, receive personalized crop
recommendations, and find nearby agricultural markets through a single integrated
workflow. The system also provides role-based administrative features to manage users,
crop data, market records, and market-price information.

Overall, the project improves digital access to agricultural decision support, reduces manual
effort in advisory planning, and demonstrates the practical value of combining
recommendation and location-based market guidance in one platform.

5.2 Lesson learnt/Outcome
This project marks the successful completion of Project II at NIST College, where I
developed a complete AgriNova advisory platform using Flutter and Django REST
Framework. Through this project, I gained practical experience in API-based system design,
mobile-backend integration, authentication workflows, and algorithm-driven feature
implementation.

Despite time and resource limitations, I was able to implement the essential system
features. Users can register and log in, submit soil data, get ranked crop recommendations,
find nearest markets, and review recommendation and search history from a unified mobile
interface. The admin can efficiently manage key datasets from controlled backend modules.

Key outcomes include:
Improved Decision Support: Users can make more informed crop planning choices using
soil-based recommendation rather than guess-based practice.
Better Market Accessibility: Users can quickly discover nearby markets through
coordinate-based search, helping reduce selling friction.
Streamlined Data Management: Administrators can maintain crops, markets, prices, and
users in a structured way.
Practical Full-Stack Learning: The project strengthened my understanding of backend API
logic, mobile integration, security handling, and database-driven workflows.

5.3 Future Recommendations
To further enhance AgriNova, several improvements can be considered:
- Production-Ready Deployment: Configure strict CORS policies, secure host settings,
	environment-based secrets, and hardened server configuration for real deployment.
- Scalable Database and Infrastructure: Migrate from the current development-focused
	database setup to a production-grade database and scalable deployment architecture.
- Advanced Advisory Intelligence: Add explainable recommendation details, weather-aware
	logic, and optional market trend analysis to support stronger decision quality.
- Improved User Engagement: Add localized language support, notification workflows,
	and richer advisory dashboards for better usability.
- Offline-First Mobile Experience: Add local caching and sync mechanisms so users in
	low-connectivity regions can still use key advisory features reliably.

By incorporating these enhancements, AgriNova can become more scalable, secure,
and user-friendly, and can deliver a stronger long-term impact for digital agriculture
advisory services.
